`porter config` contains commands for managing local CLI configuration settings.

---

## `porter config`

Display the current CLI configuration, including project, cluster, and authentication details.

**Usage:**
```bash
porter config
```

**Flags:**

| Flag | Description |
|------|-------------|
| `--show-token` | Display the CLI authentication token in the output. By default, the token is hidden for security. Pass this flag to reveal the token value. |

<Info>
Configuration values can be overridden using flags or environment variables. See [Basic Usage](/cli/basic-usage) for details.
</Info>

---

## `porter config set-project`

Set the active project. This command shows a list of projects you have access to and lets you select one.

**Usage:**
<CodeGroup>
```bash Interactive Selection
porter config set-project
```

```bash Direct ID
porter config set-project 12345
```
</CodeGroup>

---

## `porter config set-cluster`

Set the active cluster. This command shows a list of clusters in the current project and lets you select one.

**Usage:**
<CodeGroup>
```bash Interactive Selection
porter config set-cluster
```

```bash Direct ID
porter config set-cluster 67890
```
</CodeGroup>

<Tip>
Run `porter cluster list` first to see available clusters and their IDs.
</Tip>

## `porter config set-login-method`

Select which authentication flow `porter auth login` uses. The choice is persisted in your local config so future logins use the same method.

**Usage:**
```bash
porter config set-login-method [v1|v2]
```

**Arguments:**

| Value | Description |
|-------|-------------|
| `v1` | Porter JWT login (default). Opens a browser against your Porter instance and stores a long-lived token. Use this in CI/CD or against Porter instances without WorkOS configured. |
| `v2` | WorkOS session login. Runs an OAuth 2.0 device authorization flow, stores a short-lived access token and a refresh token, and refreshes transparently. Sessions appear in the **CLI sessions** list in your Porter account settings and can be revoked from the dashboard. Requires that your Porter instance has WorkOS configured. |

<CodeGroup>
```bash Switch to WorkOS sessions
porter config set-login-method v2
porter auth login
```

```bash Switch back to Porter JWT
porter config set-login-method v1
porter auth login
```
</CodeGroup>

<Info>
See [`porter auth login`](/standard/cli/command-reference/porter-auth#porter-auth-login) for a detailed comparison of the two flows.
</Info>

## Configuration File

Porter stores configuration in `~/.porter/config.json`. You can also override settings using environment variables:

| Variable | Description |
|----------|-------------|
| `PORTER_PROJECT` | Override project ID |
| `PORTER_CLUSTER` | Override cluster ID |
| `PORTER_TOKEN` | Override authentication token |
| `PORTER_HOST` | Override API host |
