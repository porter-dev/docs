`porter env` contains commands for managing environment variables and secrets.

## Prerequisites

- You've logged in to the Porter CLI after running [porter auth login](/standard/cli/command-reference/porter-auth)
- You're connected to the correct project by running [porter config set-project](/standard/cli/command-reference/porter-config)
- You're connected to the correct cluster by running [porter config set-cluster](/standard/cli/command-reference/porter-config)

---

## `porter env pull`

Pull environment variables from an application or environment group to your local machine.

**Usage:**
```bash
porter env pull [flags]
```

**Options:**

| Flag | Short | Description |
|------|-------|-------------|
| `--app` | `-a` | Pull variables from an application |
| `--group` | `-g` | Pull variables from an environment group |
| `--file` | `-f` | Output file path (default: `.env`) |

<CodeGroup>
```bash From Application
porter env pull -a my-app
```

```bash From Environment Group
porter env pull -g production-secrets
```

```bash To Custom File
porter env pull -a my-app -f .env.local
```
</CodeGroup>

<Info>
This creates a `.env` file in your current directory with all the environment variables.
</Info>

## `porter env set`

Set environment variables and secrets for an application or environment group.

**Usage:**
```bash
porter env set [flags]
```

**Options:**

| Flag | Short | Description |
|------|-------|-------------|
| `--app` | `-a` | Target application |
| `--group` | `-g` | Target environment group |
| `--variables` | | Comma-separated key=value pairs for plain variables |
| `--secrets` | | Comma-separated key=value pairs for secret variables |

<CodeGroup>
```bash Set App Variables
porter env set -a my-app --variables KEY1=value1,KEY2=value2
```

```bash Set Group Variables
porter env set -g my-env-group --variables KEY1=value1
```

```bash Set Secrets
porter env set -g my-env-group --secrets API_KEY=secret123,DB_PASSWORD=pass456
```

```bash Set Both
porter env set -g my-env-group \
  --variables NODE_ENV=production,LOG_LEVEL=info \
  --secrets DATABASE_URL=postgres://...
```
</CodeGroup>

<Warning>
Use `--secrets` for sensitive values like API keys and passwords. These are encrypted and not visible in logs.
</Warning>

## `porter env unset`

Remove environment variables from an application or environment group.

**Usage:**
```bash
porter env unset [flags]
```

**Options:**

| Flag | Short | Description |
|------|-------|-------------|
| `--app` | `-a` | Target application |
| `--group` | `-g` | Target environment group |
| `--variables` | | Comma-separated list of variable names to remove |

<CodeGroup>
```bash Unset from App
porter env unset -a my-app --variables KEY1,KEY2
```

```bash Unset from Group
porter env unset -g my-env-group --variables OLD_KEY,UNUSED_VAR
```
</CodeGroup>
